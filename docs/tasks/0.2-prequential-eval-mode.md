# Task 0.2 — Prequential / online eval mode + inference-FLOP accounting

- Status: BLOCKED on 0.1
- Depends on: 0.1 (FLOP counter, harness, leaderboard)
- Branch from: `master` after 0.1 merges

## Context
Read ADR 0004 and `docs/learning/concepts/prequential-evaluation.md`. Adds the real metric on
top of the amortized baseline harness.

## Scope
- Prequential eval loop: for each byte of the eval stream, predict **before** reveal (accumulate
  bits), then optionally adapt; cumulative bpb over the stream.
- **Total-FLOP accounting**: count pretraining + prediction + any test-time-adaptation FLOPs in
  one budget. Extend the FLOP counter to inference/adaptation paths.
- Data carve: enwik8 byte-level, **final 5 MB = fixed eval stream** (never in prior corpus),
  first ~95 MB = free prior corpus. CI uses a scaled `text8` clone, identical protocol.
- Leaderboard reports **bpb-vs-total-FLOP curves** (multiple budgets).
- Re-run the transformer baseline under prequential to produce the first official curve.

## Acceptance criteria
- Tests: no eval-stream leakage into prior corpus; adaptation FLOPs are actually counted
  (a model that adapts more shows higher FLOPs); prequential bpb matches a hand-computed tiny case.
- Gates green; offline smoke run produces a curve + plot.
- `docs/learning/experiments/` entry: the baseline prequential curve.

## Deliverable
Own branch + own PR (do not merge); cross-reviewed by a different vendor.
