## Intuition

Backprop learns by pushing one global error signal backward through every layer. **Predictive
coding (PC)** is more local and more *dynamic*: each layer holds a guess about the layer below, the
mismatch between guess and reality is a **prediction error**, and the network **settles** —
iteratively nudging its internal latents to shrink those errors — before it commits to an answer.
"Inference" is not one forward pass; it is a short optimization the model runs *at use-time*.

That dynamism is also PC's per-FLOP weakness. It usually settles **uniformly** — every input gets
the same number of iterations — so it pays full settling cost on trivial inputs and tends to lose on
[loss-per-FLOP](/concepts/loss-per-flop-and-scaling-laws). The idea we test in
[B.1](/experiments/B.1-surprise-gated-pc-refinement) is to make settling **surprise-gated**: spend
many iterations on the bytes the model finds hard, one or zero on the easy ones. Same total settling
budget, *concentrated where it reduces loss* — a [Source-(iv)](/concepts/source-iv-advantage) bet
about *dynamics*, not architecture.

## The math (logit-correction variant)

A frozen core gives a hidden state $h$ and base logits $\ell_{\text{core}}$. A small PC module adds
a latent $z$, a generative map $W$ (it predicts the hidden, $\hat h = Wz$), and a readout
$V$ (it emits a *correction* to the logits). Inference minimizes a free energy that trades fitting
the hidden against a latent prior:

$$F(z) = \tfrac{1}{2\sigma_h^2}\lVert h - Wz\rVert^2 + \tfrac{1}{2\sigma_z^2}\lVert z\rVert^2$$

It does so by $K$ gradient (settling) steps, warm-started from the previous byte's latent:

$$z \leftarrow z - \eta\Big[\tfrac{z}{\sigma_z^2} - \tfrac{W^\top(h - Wz)}{\sigma_h^2}\Big]$$

The refined prediction is $\ell = \ell_{\text{core}} + V^\top z$. Learning is **local and
gradient-free**: after the true byte is revealed, the readout follows the exact cross-entropy
gradient $\Delta V \propto -z(p-e)^\top$ and the generative weights follow the PC error rule
$\Delta W \propto (h-Wz)z^\top$. No backward pass ever touches the core.

The **gate** reads a *pre-reveal* surprise proxy — how flat the current prediction is,
$1 - \max p$ — and maps it to the settling depth $K$, **calibrated so the average $K$ equals the
uniform baseline's**. So the win can only come from *allocation*, never from spending more.

<Callout variant="note" title="why the proxy must be pre-reveal">
The gate decides $K$ *before* the byte is shown, from the model's own confidence — never from the
answer. That is the [prequential](/concepts/prequential-evaluation) discipline: a model that peeked
at the truth to decide how hard to think would be cheating the future with the past.
</Callout>

## Worked example (why it can — and here doesn't — pay off)

Start $V = 0$, so the correction is zero and the model is *exactly* the core (this bounds the worst
case — a confident-wrong module can never starve the truth at step one). As bytes stream by, the
readout learns which latent directions predict which bytes; on a *hard* byte the gate grants more
settling steps, $z$ moves further toward explaining $h$, and the learned readout turns that into a
sharper logit — *if* the core's confidence actually varies byte to byte.

That "if" is load-bearing, and [B.1 measured it](/experiments/B.1-surprise-gated-pc-refinement). On
an order-0 synthetic corpus a lightly-trained core is *uniformly* unsure, so the surprise proxy
barely moves and the gate degenerates to the uniform schedule. Even once the core is trained enough
to be differentially confident, the PC layer here costs more than it returns:

## Surprise-gating, read honestly

The gate *direction* is confirmed — gated settling beats uniform settling at identical eval FLOPs —
but on this corpus the whole mechanism is **Pareto-hollow**. The lesson is not "PC fails"; it is
that "spend more compute on hard bytes" only means something when the data *has* hard bytes.

<Callout variant="caveat">
The gate needs a **trained core to bite** and **data with genuine difficulty variation** to be
worth biting. Both prerequisites are structural, not tuning details — and order-0 synthetic text
has neither. A fair test of the lever wants real `enwik8`, where per-byte difficulty actually
varies. Read [B.1](/experiments/B.1-surprise-gated-pc-refinement) for the controlled measurement.
</Callout>
