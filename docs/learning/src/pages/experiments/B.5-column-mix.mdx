## The bet (Source-(iv))

[`delta_mix`](/experiments/B.4-delta-mix) is already a **sheet of one**: a single online delta-rule
fast-weight $W$ over a sparse signed-hashed bag of byte n-grams, mixed with the count ladder. Its
prediction $z = W\varphi$ is **linear in $\varphi$**. `column_mix` replaces that one $W$ with $C$
**columns**, one chosen per byte by a cheap learned gate — an **MoE-of-deltas**, 100% local learning, no
global backward pass.

The seductive pitch is **"$C$ columns = $C\times$ capacity at constant FLOPs."** It is **false**. A sparse
read is $O(sV)$ regardless of the table width $d$ (the bar's own feasibility crux), so `delta_mix` can
simply set $\texttt{delta\_dim} = C\cdot d$ for **zero** extra FLOPs and get the *same* capacity in **one
shared $W$** — updated by **every** byte (no data split), fewer collisions, full convergence. So a sheet
buys no FLOP-free capacity the bar cannot buy more cheaply.

<Callout variant="insight" title="the only real lever is route-conditional selection">
A single linear $W$ computes one fixed function of $\varphi$: the same features always map to the same
logits. A routed sheet computes $z = W_{\text{route}(\text{ctx})}\,\varphi$ — it **switches weight
matrices on a context bucket**, so the *same* n-gram features can map to *different* logits depending on a
separate route context. That is a **multiplicative context×feature interaction** a linear-in-$\varphi$
predictor **cannot represent at any $d$**. The whole (iv) bet rides on whether real byte streams contain
that interaction *and* whether extracting it beats the capacity you gave up to partition the table.
</Callout>

## Mechanism — one chosen column per byte, plus a bandit gate

`column_mix` inherits `delta_mix`'s warmed hashed count ladder, the logistic mixer, the feature map
$\varphi$, and **all** of its FLOP accounting. It adds two pieces:

- **A column array** $W_{\text{cols}} \in \mathbb{R}^{C\times V\times d}$ (init $0$ — an unwritten column
  abstains, exactly like the bar's unwritten $W$). The chosen column **is** the bar's single delta stream.
- **A router + a per-arm bandit gate.** A cheap Fibonacci hash of the recent bytes picks a bucket $b$;
  the gate's argmax (with a little $\varepsilon$-exploration) picks the active column $c^\*$. The chosen
  column reads $z = W_{c^\*}\varphi$ (still touching only $s$ columns of one matrix) and feeds it in as the
  **same single mixer row** `delta_mix` uses — so the count-side path and mixer width are **identical** to
  the bar. After the reveal, two local rules fire: the column's own **delta (LMS)** update on its routed
  sub-stream, and an **EMA bandit** nudging $\texttt{gate}[b, c^\*]$ toward the observed reward
  ($=-\text{bits}$). The router is $O(C)$ — under 0.2% of the bar's per-byte FLOPs at $C \le 16$.

The degenerate cases are exact by construction: $C = 1$ is **bit-identical** to `delta_mix`, and gate-off
($\texttt{gate\_lr}=0,\ \varepsilon=0$) is an exact static-routing baseline — so the kill-test compares the
*same code* with routing on and off.

## The lever provably EXISTS

On a synthetic **interaction source** — a regime byte selects *which* (n-gram → next-byte) mapping is
active, so the same n-gram predicts **different** bytes in different regimes — the routed sheet ($C>1$)
beats the single column, **and** the single column does **not** catch up even when its
$\texttt{delta\_dim}$ is grown to the sheet's full width $C\cdot d$:

<Callout variant="insight" title="selection beats capacity — when the interaction is real">
$C>1$ beats $C=1$ on the interaction source by a clean numeric margin, **and** $C=1$ plateaus *even at
matched capacity* ($\texttt{delta\_dim}=C\cdot d$). So the interaction is real and capacity cannot fake
it: route-conditional selection extracts loss a single linear-in-$\varphi$ predictor is structurally blind
to. The lever is not imaginary — the only question left is whether **real text** rewards it.
</Callout>

## …but it does NOT pay — the matched-FLOP kill-test

The kill-test runs four entrants on real `enwik8` (4 MB slice) at **matched total FLOPs**
($\approx 1.07\times10^{10}$, $C = 4$). Routing must earn its keep as **selection**, so it must beat
**both** the bar **and** the matched-*capacity* control:

| run | bpb | what it isolates |
| --- | ---: | --- |
| (a) `delta_mix` — the bar ($C=1$) | 2.4376 | one shared delta stream, every byte |
| (b) `column_mix` — learned gate | 2.4577 | $C=4$ routed columns + an online bandit gate |
| (c) `column_mix` — gate-off (hash route) | 2.4427 | $C=4$, fixed hash route (the routing, no learning) |
| **(d) `delta_mix` — matched capacity ($\texttt{delta\_dim}=C\cdot d$)** | **2.4181** | the columns' total width in **one shared table** |

**Verdict: KILL.** The best routed config (gate-off, **2.4427**) is above **both** the bar
(**2.4376**) and the matched-capacity control (**2.4181**). The same FLOPs spent on **one bigger shared
table** beat **partitioning** that capacity across routes — *capacity beats selection*. And the
**learned** gate (b, **2.4577**) is **worse than gate-off** (c, **2.4427**): the bandit added noise rather
than finding a better-than-hash assignment.

### Why it failed — the diagnostics

The pre-registered failure mode is exactly what happened: at matched total FLOPs each of the $C$ columns
sees only $\approx N/C$ delta updates, so every column is $\sim C\times$ less converged. The load was
**perfectly even** — no column specialized:

The delta-row mixer weight stayed load-bearing (the columns still predict), but no column reached a
distinct conditional bpb against the bar on its own routed bytes — there was no per-column specialization
for the gate to find. A cross-vendor (gpt-5.5) audit of the router gather/argmax and the bandit-update
scalar count found **no FLOP undercharge**, so the kill rests on an honest, matched comparison.

## What we learned

1. **Capacity beats selection on dense byte targets.** Every byte carries a target, so the predictor that
   sees *all* the data in *one* well-converged table beats $C$ partitioned, half-trained columns — even
   when a context×feature interaction genuinely exists. Spend the FLOP-free capacity, don't partition it.
2. **A real lever can still be Pareto-hollow.** The selection interaction is provably present (the
   synthetic source isolates it), yet on real text its marginal loss-reduction is swamped by the
   convergence and capacity it costs. *Existing* ≠ *paying* — the [first-finding](/experiments/first-finding-pareto)
   reflex, carried into routing.
3. **The learned gate underperformed gate-off.** With no specialization to discover, the bandit's
   exploration only injected noise — a clean reminder that a learned router is a liability unless there is
   structure for it to find.
4. **This is why the local-learning thread turns to control.** On dense next-byte prediction, capacity and
   convergence dominate, leaving little room for structure to pay. The scalar-reward,
   no-derivative [control setting](/concepts/in-context-control) is where a local learner's
   credit-assignment edge has a real chance — so the search moves there next.
