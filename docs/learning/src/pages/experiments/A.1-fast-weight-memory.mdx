## The bet (and why it fails here)

Gradient descent is an expensive way to **memorize**. A
[fast-weight associative memory](/concepts/fast-weight-memory) memorizes with a single
gradient-free outer-product write ŌĆö $O(1)$, no backward pass. The bet: hand rote memorization to
the cheap memory, let the gradient core do generalization, and the gradient FLOPs buy more
loss-reduction per FLOP ŌĆö a real [Source-(iv)](/concepts/source-iv-advantage) claim.

**Measured, the bet does not hold on this corpus.** Three facts kill it:

1. **No literal repetition to monetize.** The 512-byte clone eval stream has **0 repeated 6-grams**
   and **0 repeated 8-grams** (only 4 repeated 4-grams). Nothing for a rote memorizer to recall.
2. **No division of labor.** Gate weight and recall accuracy are essentially identical whether the
   core is trained ($10^{10}$) or untrained: gate mean $a$ $=$ **0.254 vs 0.255**, recall@1 $=$
   **10.8% vs 11.2%**. The memory does the same weak thing regardless of core quality.
3. **At budget 0 there are zero gradient FLOPs to "free."** Yet the memory still "wins" there ŌĆö
   because it is acting as a fixed weak always-on predictor, not as a memory freeing the budget.

So the observed crossover is fully explained by **"a weak always-on frequency-like predictor helps
a bad core and hurts a good one"** ŌĆö not by free memorization.

## What we learned

- **"Free memorization" needs literal repetition the data must contain.** On a non-repetitive
  corpus the memory degenerates to a weak always-on predictor ŌĆö the tell is gate/recall stats being
  identical trained-vs-untrained (no division of labor occurred).
- **The honest competitor is a free online unigram, not the transformer.** Beating a
  badly-under-trained transformer is not a Source-(iv) result when a $\sim10^{5}$-FLOP unigram beats
  both. Every candidate from here is plotted against that floor.
- **A good core is the memory's enemy, on real data too.** Once the transformer learns the
  structure, the crude store only adds crosstalk (the `enwik8` loss of +0.59 exceeds the clone's +0.24).
- **Anisotropy & bounded mixing are real engineering, but they buy robustness, not a win.** Keying
  on the *centered* residual makes recall possible (raw hidden states are near-parallel, adjacent
  cosine Ōēł 1.000); confidence-gated probability mixing keeps a wrong recall from exploding the bpb ŌĆö
  but neither manufactures loss-reduction the data doesn't afford.

<Callout variant="insight">
Do not read the low-budget transformer win as a Source-(iv) result. The win is **Pareto-hollow**
(the free unigram dominates the win regime) and **data-gated** (0 repeated n-grams here). That is
the [first finding](/experiments/first-finding-pareto).
</Callout>
