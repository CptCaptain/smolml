## The bet (Source-(iv))

Every prior fast-weight / PC attempt died the same way: it *refined an already-good cheap learner*, and
a free/cheap online baseline dominated it per FLOP. So this candidate attacks a **structural weakness of
the count mixer itself**, not its compute budget.

A count table is a *degenerate associative memory* — a **one-hot-key Hebbian** store, where each exact
context owns a cell bumped by `+1`. Two hard limits follow:

1. **Zero generalization.** A never-seen k-gram makes that order *abstain*; at scale most high-order
   contexts are novel.
2. **Only $K$ global mixer weights** combine the orders — "order-3 is worth $w_3$ *everywhere*." The
   mixer cannot learn that *some* trigrams are far more predictive than others.

`delta_mix` adds **one** online predictor whose learning rule is an **error-correcting delta (LMS)
update** on a *distributed* key, so it holds $d \times V$ **per-feature, per-byte** affinities: it learns
"`tio` → `n` is a strong rule" once and shares it across every context containing `tio`, weighted by its
*measured residual utility*. It also reaches **beyond the order-`max_order` ladder** (orders 7–8 in the
key, at ~free marginal cost).

## Mechanism — one extra stream on the warmed hashed ladder

It rides on the [B.3](/experiments/B.3-hashed-mix-full-corpus) warmed hashed count ladder and adds a
single learned stream:

- **Key** $\varphi(\text{ctx})$: for each n-gram order in `delta_orders` (default 3–8), hash the suffix
  n-gram (the same Fibonacci hash the count tables use) to a bucket in $[0, d)$ with a **±1 sign** from a
  second hash (signed feature-hashing — collisions cancel in expectation). $\varphi$ is $s$-sparse.
- **Predict** $z = W\varphi$ (touches only the $s$ active columns) — one more **raw-logit** row in the
  existing logistic mixer.
- **Learn** (deferred to the next byte, so there is no leakage):
  $\Delta W_{:,j} = -\eta\,\varphi_j\,(\mathrm{softmax}(z) - \mathrm{onehot}(b))$ over the $s$ touched
  columns. This is **error-correcting**, *not* plain Hebbian (a superposed-key Hebbian write collapses to
  the byte marginal — the A.1 death; an ablation confirms delta beats it).

<Callout variant="note" title="feasibility crux (FLOP honesty)">
Sparsity makes both the read and the rank-1 write $O(sV)$, **not** $O(dV)$ — a dense key would be
~5600× the bar and dead on arrival. The table size $d$ (default $2^{18}$, ~268 MiB) costs only RAM and
collision noise, **never FLOPs**. The add-on is ≈ **8.2k FLOPs/byte** on the bar's ~15.7k; every op is
charged via `smolml.flops` (the cross-vendor review caught two real discrepancies — a sign-hash
overcharge on the unsigned path and a colliding-column write that did not accumulate; both fixed).
</Callout>

### Why it is a real (iv) claim — honestly

It *is* online gradient descent, but the model is **linear in a fixed feature map** and the loss is
**convex**, so the exact gradient is $(p - \text{target}) \otimes \varphi$ — a single rank-1 outer
product with **zero backward pass, no chain rule, no 2× tax, no bad minima**. That is the legitimate
"more loss-reduction per FLOP than SGD-on-a-net" edge; the win over the *count mixer* is the per-feature
generalization above. Contrast [A.1](/experiments/A.1-fast-weight-memory): the same fast-weight family,
but a Hebbian write bolted onto a transformer — see
[the two write-rule lineages](/concepts/fast-weight-memory).

<Callout variant="insight" title="the recurring reflex, finally cleared">
**Beating an internal variant is not a per-FLOP win — you have to beat the cheap baseline at matched
FLOPs.** That reflex — plot the cheap baselines first — exposed every false win in this project: A.1,
B.1, and B.2-gated all looked good against an internal control and then died against the free/cheap
frontier. `delta_mix` is the **first** Space-B candidate to clear it.
</Callout>

## Result — the matched-FLOP kill-test (real enwik8, 4 MB slice)

The kill-test plots the cheap baselines first and runs them **at matched total FLOPs**
($\approx 1.07\times10^{10}$): the candidate must beat **both**, or it is hollow.

| (a) `counts_only` (hashed order-6) | 2.4353 | $1.050\times10^{10}$ | the cheap ladder at budget |
| **(b) `delta` (counts + delta)** | **2.4181** | $1.074\times10^{10}$ | the candidate |
| (c) `counts_more_warm` (hashed order-6) | 2.4327 | $1.072\times10^{10}$ | the same FLOPs, all on more warm counts |

### Two mechanistic diagnostics (why it is real, not noise)

<Callout variant="insight" title="the mixer trusts it (+0.8595)">
The final learned mixer weight on the delta stream is **+0.8595** — strongly load-bearing, the
**opposite** of A.1's tell (there the gate weight was ~identical trained-vs-untrained, i.e. dead). The
ensemble *chose* to lean on the delta stream.
</Callout>

## Full-corpus carve — beats the bar

The kill-test was **CI scale** (4 MB slice). The real test is the full ADR carve — final **5 MB eval**,
first **~95 MB prior**, all FLOPs counted, against the **2.0157 bpb @ $1.48\times10^{12}$**
[B.3](/experiments/B.3-hashed-mix-full-corpus) bar. The concern was that saturating count tables would
*shrink* the delta edge. The opposite happened — it **grew**:

| run | bpb | total FLOPs | peak RAM |
| --- | ---: | ---: | ---: |
| **`delta_o6_warmfull`** (delta + counts) | **1.8485** | $\mathbf{1.322\times10^{12}}$ | 5.5 GiB |
| `hashed_o6_warmfull` — the previous bar | 2.0157 | $1.478\times10^{12}$ | 5.0 GiB |
| `transformer` anchor | 5.4770 | $1.457\times10^{14}$ | 5.0 GiB |

<Callout variant="insight" title="the edge grew at scale — delta_mix is the new bar">
**`delta_o6_warmfull` = 1.8485 bpb @ $1.322\times10^{12}$ FLOPs strictly dominates the bar** (2.0157 @
$1.478\times10^{12}$) on **both** axes: −0.167 bpb AND fewer total FLOPs. The CI-scale edge was a modest
−0.0146; at full corpus it grew to −0.167, because as more high-order contexts go novel the exact count
tables abstain *more often* — exactly where the distributed delta key still carries signal. The first
candidate to beat the warmed-mixing bar; `delta_mix` is the new bar. The downside stayed bounded:
error-correction prevents A.1's marginal-collapse, and the mixer drives a useless stream's weight → 0.
</Callout>

## What we learned

1. **Attack the baseline's *structure*, not its budget.** Every Space-B candidate that refined an
   already-cheap learner stayed Pareto-hollow. Targeting the count mixer's two hard limits — zero
   generalization and only $K$ global weights — is what finally produced a non-hollow result.
2. **The write rule is the whole story.** Error-correcting delta (LMS) beats the Hebbian write that
   killed A.1: it stops moving a feature's weight once it predicts well, so a superposed key does not
   collapse to the byte marginal (an ablation confirms it).
3. **A distributed key generalizes where exact-match counts abstain** — the +0.8595 mixer weight and the
   3.73-vs-8.0 unseen-context split are the same mechanism seen two ways.
4. **The edge compounds with scale.** The CI-scale win was modest (−0.0146 bpb); at the full corpus it
   *grew* to −0.167 — the saturation concern was backwards. As more high-order contexts go novel, the
   exact tables abstain more and the distributed delta key carries more of the load. `delta_mix` beats
   the warmed-mixing bar outright and becomes the new bar.
