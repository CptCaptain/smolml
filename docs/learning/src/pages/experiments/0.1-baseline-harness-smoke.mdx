## Why this run exists

Task 0.1 builds the foundation: a byte-level data pipeline, an honest FLOP counter, a transformer
baseline behind a plug-in model interface, a train loop that stops at a fixed **training-FLOP**
budget, and a leaderboard. This run proves the metric is wired correctly.

## Setup

- Corpus: bundled offline sample, deterministic 90/10 train/val split.
- Two transformer baselines (pre-norm, RMSNorm, RoPE, causal), **same budget $5\times10^{10}$
  training FLOPs** (a ceiling — both stop at $\le$ budget), **same fixed eval context (64 bytes)**,
  CPU, seed 0:
  - **d32** — `d_model=32, n_layers=2` &rarr; 32,928 params, 220 steps.
  - **d64** — `d_model=64, n_layers=3` &rarr; 164,288 params, 46 steps.
- Both start from the uninformed baseline of **8.0 bpb** (uniform over 256 bytes).

## Result

| model | params | steps @ $5\times10^{10}$ FLOPs | final val bpb |
| --- | ---: | ---: | ---: |
| d32 | 32,928 | 220 | **3.37** |
| d64 | 164,288 | 46 | 3.82 |

## What the plot shows

The instructive bit: **at equal FLOPs the smaller model (d32) is lower** (3.37 vs 3.82). Each d32
step is far cheaper, so the same budget buys $\sim$5&times; more optimizer steps, and on a tiny
corpus that extra optimization wins over the bigger model's per-step capacity.

<Callout variant="insight">
This is exactly why the metric is *per FLOP* and read off a **curve**, not from a single model at a
single size: "bigger" is not automatically "better per FLOP," and a one-budget snapshot can
mislead. (See [loss-per-FLOP & scaling laws](/concepts/loss-per-flop-and-scaling-laws).)
</Callout>

<Callout variant="caveat">
Visualize honestly: this is a 5 KB sample, so the bigger model is also data-starved here. The
lesson is methodological (how to read bpb-vs-FLOP curves), not a finding about transformer scaling.
</Callout>

## Reproduce

```bash
uv run smolml train --data sample --d-model 32 --layers 2 --seq-len 64 \
    --eval-seq-len 64 --budget 5e10 --run-name baseline-d32
uv run smolml train --data sample --d-model 64 --layers 3 --seq-len 64 \
    --eval-seq-len 64 --budget 5e10 --run-name baseline-d64
uv run smolml leaderboard --runs-dir runs
```
