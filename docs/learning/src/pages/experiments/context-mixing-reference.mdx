## Hypothesis

A handful of order-$k$ byte models mixed by online-learned logistic weights (zero pretraining)
should reach far lower bpb than an *untrained* transformer at a tiny fraction of the FLOPs — the
Hutter-Prize lineage, reproduced at toy scale.

## Setup

- `model=context_mixing`, prequential / total-FLOP protocol, **zero pretrain** (transductive).
- Corpus: bundled English `sample` (offline), final **800 bytes** as the eval stream.
- Sweep `max_order` $\in \{0,1,2,3\}$; contrast $=$ one untrained (`pretrain_budget=0`) transformer
  (d=32, L=2, ctx=64) on the same stream. Seed 0, CPU.
- All per-byte compute charged via the non-matmul `pointwise` / `gather` primitives.

## Result

| run | bpb | total FLOPs |
| --- | ---: | ---: |
| context_mixing order 0..3 (reference) | **4.1733** | $7.02\times10^{6}$ |
| context_mixing order 0..2 (reference) | 4.2284 | $5.74\times10^{6}$ |
| context_mixing order 0..1 (reference) | 4.3922 | $4.28\times10^{6}$ |
| context_mixing order 0..0 (reference) | 4.7066 | $2.66\times10^{6}$ |
| transformer (untrained, contrast) | 7.9786 | $3.48\times10^{9}$ |


<Callout variant="caveat" title="two reference runs, don't conflate them">
The [first-finding leaderboard](/experiments/first-finding-pareto)'s context-mixing reference
(4.78 bpb @ $4.28\times10^{6}$ FLOPs) is a **different run** — the 512 B synthetic-clone tail at
the harness default order, not this 800 B English-sample order sweep. They land at near-identical
total FLOPs by coincidence (different stream length *and* max-order), so the bpb differs; each page
is faithful to its own run.
</Callout>

## Verdict

Reference ceiling established. Single-pass online mixing reaches $\approx$4.17 bpb for
$<10^{7}$ FLOPs; the untrained transformer needs $3.5\times10^{9}$ FLOPs to manage $\approx$8.0 bpb.

## What we learned

- **The non-matmul FLOP primitives matter.** The reference's per-byte cost (order-3 averages
  $\sim$8.8k FLOPs/byte, steady-state $\sim$9.5k) is real, charged work — and charged *exactly* for
  the branches each step runs, not a constant over-estimate.
- **Corpus shape, not just model.** On this real-English sample higher orders genuinely help
  (bigram/trigram structure); on the synthetic `text8` clone, whose letters are i.i.d. within
  words, orders $>0$ add $\sim$nothing — an honest reminder that the curve's shape is a property of
  the corpus.
- **A trained transformer would beat this bpb**, but only by spending orders of magnitude more
  pretraining FLOPs — exactly the per-FLOP gap a [Source-(iv)](/concepts/source-iv-advantage)
  candidate is meant to close.
