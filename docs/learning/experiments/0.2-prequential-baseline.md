# Experiment 0.2 — first prequential transformer baseline curve

**Status:** the first curve under the *real* metric (ADR 0004). **Researcher
handoff** for the docs-builder — plain markdown, intuition + what the curve shows.
Offline tiny clone, not enwik8: shapes and mechanics matter, absolute bpb does not.

## Why this run exists

Task 0.1 measured **amortized** loss-per-FLOP (train, then a held-out val pass).
Task 0.2 switches to the protocol we actually believe in: **prequential** —
predict every byte *before* it is revealed, pay −log₂ p bits, then optionally
adapt; score cumulative bpb against **total** FLOPs (pretraining + inference +
adaptation). This run is the first transformer point on that curve: pretrain on
the prior corpus to a budget, then **frozen** prediction over the eval stream.

## Setup

- Data: offline `text8`-style clone, carved per ADR 0004 — the **final 512 bytes
  are the fixed prequential eval stream** (scored one byte at a time, never
  trained on); the preceding prefix is the prior corpus.
- Model: transformer baseline, `d_model=48, n_layers=3` (95,568 params), CPU,
  seed 0, frozen during eval (adaptation off).
- Four total-FLOP budgets, driven by the pretraining allowance: 0 (no pretrain),
  2×10⁹, 10¹⁰, 4×10¹⁰. Total FLOPs = pretrain + Σ decode_step (the ~1.7×10⁸
  inference cost of decoding 512 bytes is always counted).

## Result (cumulative prequential bpb vs total FLOPs)

| pretrain budget | total FLOPs | prequential bpb |
| --- | ---: | ---: |
| 0 (transductive, untrained) | 1.7×10⁸ | 8.00 |
| 2×10⁹ | 1.6×10⁹ | 7.69 |
| 10¹⁰ | 1.0×10¹⁰ | 6.01 |
| 4×10¹⁰ | 3.9×10¹⁰ | **4.21** |

## What the curve shows

The leaderboard plot is **final bpb (y) vs total FLOPs (x, log scale)** — one
point per run (a budget-sweep curve), prequential dashed (amortized solid). The
four points descend monotonically: spend more compute (here, more pretraining) and
the model is less surprised by each new byte. The **zero-pretrain point sits at
≈8.00 bpb (measured 8.0003)** — an **untrained, frozen "no-model" anchor** (only
byte 0 uses the exact uniform prior; the rest come from the random-init model, so
it lands a hair above 8.0). Its non-zero x-position (1.7×10⁸) is the inference cost
of decoding the stream, which the metric refuses to give away for free.

Two ideas this makes concrete:

- **Total-FLOP honesty.** Unlike the amortized curve (x = training FLOPs only),
  here *every* FLOP counts, flowing through one channel: each per-byte `step`
  returns the prediction **and** all FLOPs it spent (inference + any adaptation),
  so compute cannot hide at eval. This baseline is frozen, so adaptation FLOPs are
  0; a future fast-weight candidate will spend them on the same budget.
- **One axis will unify the spectrum.** Amortized (all FLOPs up front, then frozen)
  is one end. Genuine **transductive** (zero pretraining, *learning online*) and
  **hybrid** points are not shown here — the frozen anchor never learns online —
  and await an adapting candidate. Once it exists, all are points on this one
  curve, compared by lowest bpb per total FLOP.

Caveat to visualize honestly: 512 bytes of a synthetic clone is a mechanics
check, not a scientific bpb. The lesson is the protocol (predict-before-reveal,
total-FLOP accounting), not the numbers.

## Worth visualizing (for the docs-builder)

## Reproduce

```bash
for b in 0 2e9 1e10 4e10; do
  uv run smolml prequential --data synthetic --synthetic-bytes 200000 \
      --eval-bytes 512 --context-window 512 --pretrain-budget $b --d-model 48 \
      --layers 3 --seq-len 128 --run-name preq-b$b
done
uv run smolml leaderboard --runs-dir runs
```
